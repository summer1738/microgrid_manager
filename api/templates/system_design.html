1. Why user registration is needed

Registration allows the system to: identify users, assign the correct microgrid package, configure based on energy setup, and manage different scales.

UserSystem type
HouseholdSmall solar system
ClinicMedium microgrid
Business parkLarge microgrid
2. System packages
Household package

Single home, small PV (1–5 kW), small battery, few appliances.

  • PV forecasting
  • Battery monitoring
  • Energy usage dashboard
Community microgrid

3–10 homes, shared solar & battery.

  • Load sharing
  • Energy allocation
  • Multi-user consumption monitoring
Institutional / critical facility

Clinic, school, rural business center.

  • Priority load management
  • Energy scheduling
  • Reliability & backup power
3. Registration steps
  1. Create account — name, email, password, location
  2. Choose system type — Household / Community / Institutional
  3. Enter system details — number of buildings, PV capacity (kW), battery capacity (kWh), critical loads
  4. Dashboard configures — features enabled by package (see table below)

6. How this helps the AI model

Forecasting and control adapt to the setup:

  • Household: Forecast PV → manage battery → supply home
  • Community: Forecast PV → distribute power to houses
  • Institutional (e.g. clinic): Forecast PV → prioritize medical equipment & critical loads

Scalability: The same system supports small solar, medium microgrids, and large institutional systems — strong system design.
